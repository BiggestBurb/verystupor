About Vocai

A browser-based hub for playing preserved Flash and HTML5 games — pulling directly from archive.org and GitHub, with nothing installed and no server required.

01What is Vocai?

Vocai is a front-end only game hub that runs entirely in your browser. It has no backend, no database, and no server. Every game is either loaded from a folder on your device or fetched live from a public source like archive.org or a GitHub repository.

The goal is to make preserved games as easy to browse and play as possible — with a fast grid interface, persistent history, and a settings system — without needing to download anything or use a separate emulator. Ruffle (the community-built Flash emulator) handles all SWF files directly in the browser. HTML games open in a fullscreen iframe.

03How to Upload

Step by Step

  1. Click the UPLOAD button in the sidebar to open the file panel.
  2. On the Upload tab, either click the drop zone or drag a folder onto it.
  3. Your browser will read all .swf and .html files from the folder recursively.
  4. The vault grid will populate with every game found. Click any card to play.

SWF Files

Played via Ruffle, the open-source Flash emulator. The file is read into memory and passed to Ruffle — no internet required once the folder is loaded.

HTML Files

Opened via FileReader — the raw HTML is written into a new about:blank window. Works best for fully self-contained HTML files with no external assets.

04Archive System

How archive.org Works Here

Every archive.org collection has a metadata API endpoint at archive.org/metadata/IDENTIFIER. This returns a JSON object listing every file in that collection. Vocai calls this endpoint, filters for .swf and .html files, and builds a play URL for each one using archive.org's CORS-enabled download server.

SWF Play URL Format

https://cors.archive.org/cors/IDENTIFIER/filename.swf — This endpoint adds the correct CORS headers so Ruffle can fetch and run the file cross-origin.

HTML Play URL Format

https://archive.org/download/IDENTIFIER/filename.html — Archive.org serves the file directly. Since all assets in the same archive folder share the same base URL, relative paths (scripts, images, styles) resolve correctly inside the iframe.

Session Persistence

The last loaded archive is saved to localStorage. On refresh, Vocai automatically re-fetches the same archive so your library reappears without needing to click anything.

05GitHub System

How GitHub Loading Works

Vocai calls the GitHub Trees API at api.github.com/repos/USER/REPO/git/trees/BRANCH?recursive=1 to get a flat list of every file in the repository. It filters for .html files, then builds a direct play URL for each one.

GitHub Pages URL

For repos with GitHub Pages enabled, files are served at username.github.io/repo/path.html with correct MIME types and no iframe restrictions.

rawgithack.com

For repos without GitHub Pages, raw.githack.com is used as a CDN proxy — it serves raw GitHub files with proper text/html MIME types so the iframe renders them correctly.

Custom GitHub Repo

In the Archives → GitHub Collections section, paste any username/repo (or full GitHub URL) and an optional branch name. Vocai will pull the file list automatically. If the repo has no .html files, it will return an error.

Rate Limiting

The GitHub API allows 60 unauthenticated requests per hour per IP. Each GitHub source load uses exactly 1 request (the tree fetch). Normal usage will never hit this limit.

06Settings Reference
Site NameChanges the title shown in the sidebar and browser tab.
Quick URL ButtonAdds a custom button below the search bar that opens any URL in a fullscreen iframe. Useful for linking to a proxy or tool.
Accent ColorChanges the primary highlight color used across the entire UI — cards, buttons, labels, waves.
Cards Per PageControls how many game cards are shown per page in the vault grid. Higher values load slower on large archives.
ThemeSwitches between Dark (default) and Light mode. Light mode forces black text on all elements.
Sidebar on StartWhether the sidebar is visible or collapsed when the page first loads.
Background WavesToggles the animated canvas wave background. Turning it off improves performance on slower devices.
Boot AnimationEnables or disables the "hello." splash screen shown on page load.
Clear LibraryRemoves all currently loaded files from the vault. Also clears the saved archive state so nothing reloads on refresh.
Clear HistoryWipes the recent files list shown in the sidebar. Does not affect loaded files.
07Error Reference
ErrorWhat It Means & How to Fix It
No .swf or .html files foundThe archive.org collection exists but contains no compatible files. Try a different archive or check the identifier is correct.
No HTML files found in repoThe GitHub repository has no .html files at the tree level, or the branch name is wrong. Double-check the branch (main vs master).
Could not reach GitHub APIThe GitHub API returned an error — usually a wrong username/repo, a private repo, or the rate limit (60 req/hr) was hit. Wait an hour or check the repo path.
No file listThe archive.org metadata API returned an empty or malformed response. The identifier may be incorrect or the collection may be private.
Pop-up blockedYour browser blocked the game window from opening. Allow pop-ups for this site in your browser settings and try again.
File not found after reloadA game in your Recent history was from an archive that no longer has that file, or the file was renamed. The archive may have changed since it was first loaded.
No archive info savedAn older history entry was saved before arc metadata tracking was added. Re-load the archive manually from the Upload panel.
08Recent History

Every game you open is saved to localStorage under the key vchist. Entries are grouped in the sidebar by their source archive or "Local Files". Clicking a recent game:

  • If the source is already loaded → opens immediately
  • If it's an archive.org source → auto-fetches the archive, then opens the file
  • If it's a GitHub source → auto-fetches the repo tree, then opens the file
  • If it's a local file → prompts you to re-upload the folder
